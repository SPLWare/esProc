When using parameters in the execute function, their positions in the SQL statement are held by question marks (?) and they are written in order after the statement. The values to be assigned by the SQL statement can be represented all or partly by parameters, as A3 and A4 respectively demonstrate. Note that a string in an esProc function should be enclosed by double quotation marks (in the form of "бн"), which is different from the SQL syntax.
