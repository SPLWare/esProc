A.to@z(i,n) function helps to facilitate the batch aggregatge of a large amount of data by making all but the last segment have the same number of members. If the number of members in each segment is far more than the number of segments, like 100 segments with each having 100,000 members, there will be at most 100099 members in the last segment, which is not so many more than the number of members in the other segments.
